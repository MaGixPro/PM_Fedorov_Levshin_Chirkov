## 1) COSMIC Function Points (CFP)

### Подход
Оценка выполнена на уровне ключевых функциональных процессов: бронирование, отмена, подтверждение, уведомления, админ-управление и отчётность.  
Для каждого процесса учитываются движения данных: **Entry / Exit / Read / Write**.

### Итог (учебная оценка)
**Итого CFP:** **~40–55** (проект среднего масштаба).

---

## 2) Use Case Points (UCP)

### 2.1 UUCW — Weighted Use Cases
Используем 12 use case (см. `Use_cases.md`).  
Классификация (учебно):
- 4 простых (5)
- 6 средних (10)
- 2 сложных (15)

**UUCW = 4×5 + 6×10 + 2×15 = 20 + 60 + 30 = 110**

### 2.2 UAW — Weighted Actors
Акторы:
- Пользователь (человек) — **3**
- Администратор (человек) — **3**
(опционально: “Сервис уведомлений” как внешний актор — **1**)

**UAW = 6** (или 7 с внешним сервисом)

### 2.3 TCF — Technical Complexity Factor
Считаем сумму факторов ΣTF и коэффициент:
`TCF = 0.6 + 0.01 × ΣTF`

(Значения факторов приведены в таблице)

**TCF ≈ 0.95**

### 2.4 ECF — Environmental Factor
`ECF = 1.4 - 0.03 × ΣEF`

(Значения факторов приведены в таблице)

**ECF ≈ 0.85**

### 2.5 Итог UCP
`UCP = (UUCW + UAW) × TCF × ECF`

**UCP = (110 + 6) × 0.95 × 0.85 ≈ 93.7 ≈ 94**

### 2.6 Трудозатраты по UCP
Принято: **1 UCP = 20 чел.-часов** (консервативно)

**94 × 20 = 1880 чел.-часов**  
В человеко-месяцах (160 ч/мес): **1880/160 ≈ 11.75 чел.-мес**

## 3) Коллективная экспертная оценка (Planning Poker)

Шкала Story Points: 1,2,3,5,8,13.  
Конверсия (учебно): **1 SP = 8 чел.-часов**.

Оценка сделана по use cases (или крупным user story).  
Итог (пример):
- Сумма **SP = 45**
- Часы = **45×8 = 360 чел.-часов**

---

## 4) Сравнение методов и выводы

- **UCP** даёт оценку всего проекта “с запасом” (учитывает сложность, факторы, инфраструктуру, тестирование).
- **Planning Poker** ближе к оценке ядра MVP и полезен для спринтов.

Рекомендуемая стратегия:
- UCP — для оценки общего масштаба и резерва времени
- Planning Poker — для планирования итераций разработки MVP

## 5) Команда и сроки

Если ориентироваться на UCP ≈ 1880 часов:
- команда 4–5 человек (dev+qa+pm частично): **2–3 месяца**

Роли (оптимально):

- QA

## 6) Что можно упростить (срез функционала)

Чтобы уменьшить масштаб MVP, можно отложить:
1) Отчёты/аналитику (UC12)
2) Жалобы/модерацию (UC11)
3) Сложные правила штрафов (часть UC10 — оставить только базовые лимиты)
4) Расширенные уведомления (оставить 1–2 типа)
